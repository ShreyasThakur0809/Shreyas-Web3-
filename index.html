Execution & Transactions

How instructions are bundled and executed differs significantly, impacting developers and users.

Solana

Transaction Model

A transaction is a bundle of one or more atomic instructions. All instructions succeed or fail together.

Instruction 1: Create a data account (PDA) for a user.
Instruction 2: Initialize data in that account.
Instruction 3: Transfer SOL to the new account for rent.

This atomicity allows for complex, multi-step operations in a single, secure transaction.

Ethereum

Transaction Object

A transaction is a single message from an EOA to another account, containing key fields.

  • To: Contract or EOA address.
  • Value: Amount of ETH to send.
  • Data: Encoded function call and arguments.
  • Gas Limit/Price: Fee parameters.
  • Nonce: Transaction counter.
  • Value: (Optional) ETH to send.

The EVM executes the code based on the `data` field, modifying the contract's internal storage.

dApp Development Concepts

Key tools and concepts for building frontends that interact with blockchains.

React Hooks: `useState` & `useEffect`

These are the building blocks of modern React applications for managing state and side effects.

`useState`

Manages component data. When state changes, the component re-renders.

`useEffect`

Handles side effects like data fetching or subscriptions after the component renders.

Example: Fetching Data

const [data, setData] = useState([]);

useEffect(() => {
  // Runs once after initial render
  fetch('api/data')
    .then(res => res.json())
    .then(json => setData(json));
}, []); // Empty array means run once

Ethereum: ABI

The <span class="font-semibold text-teal-600">Application Binary Interface (ABI)</span> is the "API documentation" for an Ethereum smart contract.

It's a JSON file that defines how to call contract functions and interpret their data, enabling tools and UIs to interact with them.

Example ABI Snippet for `add(uint a, uint b)`

{
  "inputs": [
    { "name": "a", "type": "uint256" },
    { "name": "b", "type": "uint256" }
  ],
  "name": "add",
  "outputs": [
    { "name": "", "type": "uint256" }
  ],
  "stateMutability": "pure",
  "type": "function"
}

Dive into the basics of Solidity, the primary language for writing smart contracts on Ethereum.

Solidity Basics: Version, Contract, Variables

Every Solidity file starts with a version pragma and defines contracts, which are the fundamental building blocks.

Example: Simple Contract with Variables

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0; // Version pragma

contract MyFirstContract { // Contract keyword
  uint256 public myUint; // Unsigned integer
  int256 public myInt;   // Signed integer
  bool public myBool;    // Boolean
  address public myAddress; // Ethereum address
  string public myString;  // String

  // Constructor: runs once on deployment
  constructor(uint256 _initialUint) {
    myUint = _initialUint;
    myInt = -100;
    myBool = true;
    myAddress = msg.sender; // Deployer's address
    myString = "Hello Solidity!";
  }
}

Functions: Arguments, Returns, Visibility

Functions define the logic and behavior of your smart contract. Visibility keywords control who can call them.

Example: Calculator Functions

contract Calculator {
  uint256 public currentValue;

  constructor(uint256 _initialValue) {
    currentValue = _initialValue;
  }

  function add(uint256 value) public { // public: callable by anyone
    currentValue += value;
  }

  function subtract(uint256 value) public {
    currentValue -= value;
  }

  function multiply(uint256 value) public {
    currentValue *= value;
  }

  function divide(uint256 value) public {
    require(value > 0, "Cannot divide by zero"); // require statement
    currentValue /= value;
  }

  function getCurrentValue() public view returns (uint256) { // view: reads state, no gas
    return currentValue;
  }
}

Control Flow: `require`, `if/else`, Loops

Solidity provides standard control flow mechanisms to manage logic and validate conditions.

`require` Statement

function setNumber(uint256 _number) public {

`if/else` and Loops

function categorizeNumber(uint256 _number) public pure returns (string memory) {
  if (_number < 10) {
    return "Small";
  } else if (_number >= 10 && _number < 100) {
    return "Medium";
  } else {
    return "Large";

function addNumbers(uint256[] memory _numbers) public pure returns (uint) {
  uint ans = 0;
  for (uint256 i = 0; i < _numbers.length; i++) {
    ans += _numbers[i];

}

Data Structures: Mappings & Arrays

Mappings are like hash tables, and arrays can be fixed or dynamic in size.

Mappings Example

contract NameRegistry {
  mapping(address => string) public names; // Declare a mapping

  function setName(string memory _name) public {
    names[msg.sender] = _name; // Insert into a mapping
  }

  function getName(address _address) public view returns (string memory) {
    return names[_address]; // Get from a mapping
  }
}

Arrays Example

contract ArrayExample {
  uint[3] public fixedNumbers; // Fixed-size array
  uint[] public dynamicNumbers; // Dynamic array

  function setFixedNumbers(uint _num1, uint _num2, uint _num3) public {
    fixedNumbers[0] = _num1;
    fixedNumbers[1] = _num2;
    fixedNumbers[2] = _num3;
  }

  function addDynamicNumber(uint _number) public {
    dynamicNumbers.push(_number); // Add to dynamic array
  }

  function getDynamicNumbersCount() public view returns (uint) {
    return dynamicNumbers.length;
  }
}

Structs: Custom Data Types

Structs allow you to group multiple variables of different types into a single unit.

Example: Person Struct

contract PersonContract {
  struct Person {
    string name;
    uint age;
    address addr;
  }

  mapping(address => Person) public persons; // Mapping of address to Person struct

  function setPerson(string memory _name, uint _age) public {
    persons[msg.sender] = Person({ // Create and store a struct
      name: _name,
      age: _age,
      addr: msg.sender
    });
  }

  function getPerson() public view returns (string memory, uint, address) {
    Person memory person = persons[msg.sender];
    return (person.name, person.age, person.addr);
  }
}

Data Locations: `memory`, `stack`, `storage`

Solidity has distinct data locations with different characteristics and gas costs.

`storage` (Persistent)

Persistent data on the blockchain (state variables). Costly to write.

contract StorageExample {
  uint[] public numbers; // Stored in storage

  function addNumber(uint _number) public {
    numbers.push(_number); // Modifies storage
  }
}

`memory` (Temporary, Function Scope)

Temporary data during function execution. Cheaper than storage. Used for function arguments (complex types) and local variables (complex types).

function addStrings(string memory a, string memory b) public pure returns (string memory) {
  string memory result = string(abi.encodePacked(a, b)); // 'result' is in memory
  return result;
}

`stack` (Temporary, Small Values)

Very small, temporary values (function arguments for simple types, local variables for simple types). Cleared after function execution.

function addNumbers(uint a, uint b) public pure returns (uint) {
  // 'a', 'b', and 'result' are stored on the stack
  uint result = a + b;
  return result;
}

Modifiers: Reusable Function Logic

Modifiers allow you to add reusable checks or functionality to functions.

Example: `onlyOwner` Modifier

contract Example {
  address public owner;

  constructor() {
    owner = msg.sender; // Set deployer as owner
  }

  modifier onlyOwner() {
    require(msg.sender == owner, "You are not the owner!");
    _; // Continues function execution
  }

  function setOwner(address newOwner) public onlyOwner {
    owner = newOwner;
  }
}

Returning Tuples & Pure Functions

Functions can return multiple values (tuples). `pure` functions do not read or modify state.

Example: `sumAndMul` (Pure)

function sumAndMul(uint a, uint b) public pure returns (uint, uint) {
  return (a + b, a * b); // Returns a tuple
}

Inheritance: Reusing Contract Logic

Solidity supports single inheritance, allowing contracts to inherit functionality from a base contract.

Example: `Vehicle` (Base) & `Car` (Child)

// Vehicle.sol (Base Contract)
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract Vehicle {
  string public brand;

  constructor(string memory _brand) {
    brand = _brand;
  }

  function description() public pure virtual returns (string memory) {
    return "This is a vehicle";
  }
}

// Car.sol (Child Contract)
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "./Vehicle.sol"; // Import the Vehicle contract

contract Car is Vehicle { // Inherit from Vehicle
  uint public numberOfDoors;

  // Constructor that also initializes the parent contract
  constructor(string memory _brand, uint _numberOfDoors) Vehicle(_brand) {
    numberOfDoors = _numberOfDoors;
  }

  // Override the description function to make it more specific for a car
  function description() public pure override returns (string memory) {
    return "This is a car";
  }
}

Cross-Contract Invocation (CCI) & Interfaces

Contracts interact with each other via CCI. Interfaces define a contract's external functions without implementation, enabling safe interaction.

Example: `ParentContract` calling `MessageContract`

// MessageContract.sol (The contract to be called)
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract MessageContract {
  string private message;

  function setMessage(string memory newMessage) public {
    message = newMessage;
  }

  function getMessage() public view returns (string memory) {
    return message;
  }
}

// IMessageContract.sol (Interface definition)
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IMessageContract {
  function setMessage(string memory newMessage) external;
  function getMessage() external view returns (string memory);
}

// ParentContract.sol (Calls MessageContract)
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "./IMessageContract.sol"; // Import the interface

contract ParentContract {
  address private contractAddress;

  constructor(address _contractAddress) {
    contractAddress = _contractAddress;
  }

  function setMessageFromOtherContract(string memory newMessage) public {
    // Call the external contract via its interface
    IMessageContract(contractAddress).setMessage(newMessage);
  }

  function getMessageFromOtherContract() public view returns (string memory) {
    // Get message from external contract via its interface
    return IMessageContract(contractAddress).getMessage();
  }
}

Payables: Receiving Ether

`payable` functions or addresses can receive Ether (ETH) transfers.

Example: Deposit and Withdraw

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract MyContract {
  uint public amount; // A simple variable to track deposited amount

  // This function can accept Ether
  function deposit() public payable {
    amount += msg.value; // msg.value is the amount of Ether sent with the call
  }

  // This function can send Ether
  function withdraw(address payable recipient) public {
    payable(recipient).transfer(amount); // transfer Ether to recipient
    amount = 0;
  }

  function getBalance() public view returns (uint256) {
    return address(this).balance; // Get contract's own ETH balance
  }
}
